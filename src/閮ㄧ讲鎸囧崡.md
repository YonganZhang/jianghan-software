# 中石化模型预测系统 — 部署与运行指南

## 1. 项目结构

```
software/
├── run_project.bat          # 一键启动脚本（双击运行）
├── start_project.bat        # 启动逻辑（被 run_project.bat 调用）
├── fetch_tunnel_url.py      # Cloudflare Tunnel URL 获取脚本
├── fetch_ngrok_url.py       # ngrok URL 获取脚本（备用）
├── 部署指南.md               # 本文档
├── README.md                # 接口与算法说明
├── backend/                 # 后端（Flask + Socket.IO）
│   ├── app.py               # 后端入口
│   ├── config.py            # 数据库等配置
│   ├── exts.py              # Flask 扩展初始化
│   ├── models.py            # 数据库模型
│   ├── requirements.txt     # Python 依赖清单
│   ├── instance/            # SQLite 数据库文件目录
│   │   └── wenjie.db        # 数据库（首次启动自动创建）
│   ├── blueprints/          # 路由模块
│   └── resources/julia/     # Julia 运行时（智能公式映射用）
└── frontend/                # 前端（Vue 3 + Vite + Electron）
    ├── package.json         # 前端依赖与脚本
    ├── vite.config.ts       # Vite 配置（端口、代理）
    ├── src/                 # 前端源码
    ├── dist/                # 前端构建产物（share 模式需要）
    └── node_modules/        # 前端依赖（npm install 生成）
```

## 2. 环境要求

### 2.1 必装软件

| 软件 | 版本要求 | 用途 | 下载地址 |
|------|---------|------|---------|
| Miniconda / Anaconda | 最新版 | Python 环境管理 | https://docs.conda.io/en/latest/miniconda.html |
| Node.js | 18+ | 前端开发服务器 | https://nodejs.org/ |
| Git | 最新版 | 版本管理（可选） | https://git-scm.com/ |

### 2.2 Conda 环境配置

后端需要一个包含 PyTorch（GPU）和 Flask 的 conda 环境。

**创建环境（如果没有现成的）：**

```bash
# 创建 Python 3.10 环境
conda create -n dl-gpu python=3.10 -y

# 激活环境
conda activate dl-gpu

# 安装 PyTorch（CUDA 12.1，根据显卡调整）
pip install torch==2.5.1+cu121 torchvision==0.20.1+cu121 torchaudio==2.5.1+cu121 --index-url https://download.pytorch.org/whl/cu121

# 安装后端依赖
pip install flask flask-socketio flask-sqlalchemy flask-migrate flask-compress PyJWT openpyxl pandas numpy scikit-learn matplotlib requests joblib xlsxwriter
```

**验证环境是否可用：**

```bash
conda activate dl-gpu
python -c "import torch; print('PyTorch:', torch.__version__, '| CUDA:', torch.cuda.is_available())"
python -c "import flask; print('Flask:', flask.__version__)"
```

两条命令都无报错即可。

### 2.3 前端依赖

```bash
cd frontend
npm install
```

> 国内网络建议先设置 npm 镜像：`npm config set registry https://registry.npmmirror.com`

## 3. 启动运行

### 3.1 三种运行模式

| 模式 | 命令 | 说明 | 适用场景 |
|------|------|------|---------|
| **web**（默认） | `run_project.bat dl-gpu web` | 前端 Vite + 后端 Flask，本地访问 | 日常开发、本机使用 |
| **electron** | `run_project.bat dl-gpu electron` | 桌面客户端模式 | 需要桌面应用壳 |
| **share** | `run_project.bat dl-gpu share` | Cloudflare Tunnel 公网穿透 | **发给别人远程访问** |

### 3.2 本地模式（web，推荐日常使用）

双击项目根目录的 `run_project.bat`，它会自动：

1. 检查 conda 是否可用
2. 检查指定的 conda 环境是否存在（不存在会报错并列出可用环境）
3. 启动后端（新窗口，Flask 运行在 `http://127.0.0.1:5000`）
4. 启动前端（新窗口，Vite 运行在 `http://127.0.0.1:5173`）

启动成功后，浏览器访问 `http://127.0.0.1:5173` 即可使用。

### 3.3 公网分享模式（share）

**适用场景**：需要把系统分享给不在同一局域网的人使用（如给老师/同事远程演示）。

#### 前置准备

**首次使用需安装 cloudflared：**

```bash
# Windows 命令行执行（需管理员权限）
winget install Cloudflare.cloudflared
```

安装后路径为 `C:\Program Files (x86)\cloudflared\cloudflared.exe`，不需要注册账号。

**构建前端（首次或前端代码有改动时）：**

```bash
cd frontend
npm run build
```

这会生成 `frontend/dist/` 目录，share 模式下由 Flask 直接托管前端文件。

#### 启动步骤

```
双击 run_share.bat
```

或手动执行：

```bash
start_project.bat dl-gpu share
```

启动流程：
1. 检查 conda 环境
2. 启动后端 Flask（端口 5000）
3. 等待后端就绪（约 6 秒）
4. 启动 Cloudflare Tunnel（自动建立加密隧道）
5. 打印公网链接（形如 `https://xxx-yyy-zzz.trycloudflare.com`）

#### 使用方式

```
============================================================
  PUBLIC URL: https://xxx-yyy-zzz.trycloudflare.com
  Send this link to anyone!
============================================================
```

将打印出的链接发送给对方即可。对方在浏览器中打开链接，使用以下账号登录：

> **账号：** `admin`
> **密码：** `admin`

#### 工作原理

```
对方浏览器
    │
    ▼
https://xxx.trycloudflare.com （Cloudflare CDN，全球可访问）
    │
    ▼
cloudflared（你电脑上的本地进程）
    │
    ▼
Flask :5000（你电脑上的后端服务）
    ├── /api/*     → 后端 API
    ├── /assets/*  → 前端静态资源
    └── /*         → 前端页面
```

- 不需要开放防火墙端口，不需要公网 IP
- 不需要配置代理（即使开着 Clash 也不影响）
- 不需要注册 Cloudflare 账号
- 中国大陆可正常访问（流量经 Cloudflare 亚太节点）

#### 注意事项

- 每次重启 cloudflared 会生成**新的随机域名**，需要重新发送链接
- 关闭电脑或终止进程后链接失效
- share 模式不启动 Vite 开发服务器，前端由 Flask 直接 serve
- 如果前端有改动，需要重新 `npm run build` 再启动 share 模式
- `cors_allowed_origins='*'` 仅用于演示，正式部署建议限制域名

### 3.6 手动启动（分步）

如果一键脚本有问题，可以手动分步启动：

```bash

cd backend

python app.py
# 看到 "Running on http://0.0.0.0:5000" 表示成功

# 终端 2：启动前端（web 模式）

# 看到 "Local: http://127.0.0.1:5173" 表示成功
```

### 3.7 端口说明

| 服务 | 端口 | 用途 |
|------|------|------|
| 后端 Flask | 5000 | REST API + Socket.IO |
| 前端 Vite | 5173 | 开发服务器（web 模式，代理 `/api` 到 5000） |

> share 模式下只使用端口 5000（Flask 同时 serve 前端和 API），无需 5173。

## 4. 软件迁移（发给其他人）

### 4.1 需要打包的内容

将整个 `software/` 目录压缩，但排除以下大体积/不必要的目录：

```
排除列表（不要打包）：
├── frontend/node_modules/       # 前端依赖（对方 npm install 重新装）
├── backend/dist/                # PyInstaller 打包产物
├── backend/build/               # PyInstaller 构建缓存
├── backend/instance/wenjie.db   # 数据库（首次启动自动创建）
├── backend/__pycache__/         # Python 缓存
└── backend/blueprints/__pycache__/
```

**推荐打包命令（在 software 上级目录执行）：**

```bash
# 使用 7-Zip 压缩，排除不需要的目录
7z a software.7z software/ -xr!node_modules -xr!__pycache__ -xr!dist -xr!build -xr!*.pyc
```

或手动删除上述目录后直接压缩整个文件夹。

### 4.2 接收方部署步骤

收到压缩包后，按以下顺序操作：

**第一步：解压**

将 `software.7z` 解压到任意目录（路径不要包含中文或空格），例如 `E:\software\`。

**第二步：安装基础软件**

按 [2.1 必装软件](#21-必装软件) 安装 Miniconda 和 Node.js。

**第三步：创建 conda 环境并安装依赖**

```bash
# 创建环境
conda create -n dl-gpu python=3.10 -y
conda activate dl-gpu

# 安装 PyTorch（根据显卡 CUDA 版本调整，以下为 CUDA 12.1）
pip install torch==2.5.1+cu121 torchvision==0.20.1+cu121 torchaudio==2.5.1+cu121 --index-url https://download.pytorch.org/whl/cu121

# 安装后端依赖
cd backend
pip install -r requirements.txt
```

> 如果没有 NVIDIA 显卡（无 GPU），改用 CPU 版 PyTorch：
> ```bash
> pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cpu
> ```

**第四步：安装前端依赖**

```bash
cd frontend
npm install
```

**第五步：构建前端（公网分享模式需要）**

```bash
cd frontend
npm run build
```

**第六步：启动项目**

- **本地使用**：双击 `run_project.bat`，浏览器访问 `http://127.0.0.1:5173`
- **公网分享**：双击 `run_share.bat`（或执行 `start_project.bat dl-gpu share`），将打印的链接发给对方

**第七步：登录系统**

打开页面后，使用以下账号登录：

> **账号：** `admin`
> **密码：** `admin`

## 6. 常见问题排查

### 6.1 双击 bat 脚本一闪而过

脚本末尾已有 `pause`，正常情况会停留。如果仍然闪退，右键 `run_project.bat` → 编辑，检查是否有语法错误。也可以在 cmd 中手动运行：

```bat
cd /d E:\software
run_project.bat
```

这样即使出错也能看到完整输出。

### 6.2 提示 Conda environment "xxx" not found

说明 `run_project.bat` 中配置的环境名与实际不符。运行以下命令查看已有环境：

```bash
conda env list
```

然后修改 `run_project.bat` 第 5 行的环境名，或按 [2.2 节](#22-conda-环境配置) 创建新环境。

### 6.3 页面显示"正在连接服务器..."

后端未启动成功。检查后端窗口是否有报错：

- `ModuleNotFoundError: No module named 'flask'` → 在 conda 环境中安装依赖：
  ```bash
  conda activate dl-gpu
  pip install flask flask-socketio flask-sqlalchemy flask-migrate flask-compress PyJWT
  ```
- 端口 5000 被占用 → 关闭占用程序或修改 `backend/app.py` 中的端口

### 6.4 前端 npm install 失败

国内网络问题，先设置镜像：

```bash
npm config set registry https://registry.npmmirror.com
npm install
```

### 6.5 PyTorch 安装失败或 CUDA 不可用

验证 CUDA 状态：

```bash
conda activate dl-gpu
python -c "import torch; print('CUDA:', torch.cuda.is_available(), '| Version:', torch.version.cuda)"
```

- 输出 `CUDA: False` → 检查显卡驱动是否安装，或改用 CPU 版 PyTorch
- 安装超时 → 使用国内镜像源或提前下载 whl 文件离线安装

### 6.6 share 模式：获取不到公网链接

检查 `%TEMP%\cloudflared.log` 文件中的错误信息：

- cloudflared 未安装 → 执行 `winget install Cloudflare.cloudflared`
- 后端未启动 → 确认 Flask 在端口 5000 正常运行
- 网络不通 → cloudflared 需要能访问外网（不需要代理）

### 6.7 share 模式：对方访问白屏

前端未构建。在 `frontend/` 目录下执行：

```bash
npm run build
```

构建完成后重新启动 share 模式。

### 6.8 share 模式：对方访问很慢或打不开

- `*.trycloudflare.com` 极少数情况下可能被 DNS 干扰，建议对方更换 DNS 为 `223.5.5.5`（阿里）或 `119.29.29.29`（腾讯）
- Cloudflare 自动选择最优节点，通常经香港/东京节点，延迟 < 100ms
